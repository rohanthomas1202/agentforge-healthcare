4 Verification Pipeline (4 Layers)

Every response verified deterministically (no LLM calls):

1. Drug Safety

Extracts drug names from response, checks pairs against interaction DB.

Flags: "safe to combine" when DB says interaction

2. Allergy Safety

Expands allergies to drug classes (Penicillin → amoxicillin, ampicillin...)

Flags: Drug recommended without allergy warning

3. Confidence Scorer

30% tools + 30% data richness + 20% hedging + 20% errors

Range: 0.0 - 1.0 | <0.3 = LOW

4. Claim Verifier

Extracts factual claims, checks if ≥60% of terms found in tool output.

Flags: Grounding rate <50% = hallucination

Overall Safety: drug_safety.passed AND allergy_safety.passed AND confidence ≥ 0.3 AND claims.passed

11 Key Talking Points

Why Healthcare?

  • Strongest verification story — drug safety checking is concrete and demonstrable
  • "I built a system that catches when the AI says Warfarin and Aspirin are safe together"
  • Real EHR (OpenEMR) with real FHIR R4 API, not toy data

Why LangGraph?

  • Explicit state machine with conditional routing — not "autonomous" black box
  • 10-iteration cap prevents runaway loops (critical in healthcare)
  • Native astream_events(v2) for real-time SSE streaming
  • More debuggable than CrewAI, AutoGen, or raw function calling

Verification Prevents Real Harm

  • Drug Safety: Catches contradictions — LLM says "safe" but DB says interaction
  • Allergy Safety: Catches cross-reactivity — Penicillin allergy flags amoxicillin
  • Confidence: Low scores flag empty/failed outputs before user sees them
  • Claim Grounding: Catches hallucinations — claims not found in tool output

Scaling

  • Prompt caching to avoid re-tokenizing 800-token system prompt
  • Model tiering: Haiku for simple queries, Sonnet for complex reasoning
  • SQLite → PostgreSQL for multi-instance, Redis for token pooling
  • Verification as separate microservice (CPU-bound regex)
